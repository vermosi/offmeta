# Code / UX / Security Review — 2026-03-06

## Project Overview

**offmeta** is a production Magic: The Gathering card search platform built with:
- React 19 + TypeScript 5 + Vite 7 (frontend)
- Supabase (PostgreSQL + Edge Functions on Deno)
- Tailwind CSS 4 + Radix UI
- 1,560+ tests (Vitest unit + Playwright E2E)
- 11-language i18n, PWA, AGPL-3.0 license

## 1. Security Review

### Overall Risk Level: LOW

The codebase demonstrates mature security practices. No critical vulnerabilities found.

### Strengths

| Area | Implementation |
|------|----------------|
| Authentication | JWT validation with issuer check, expiry, role hierarchy; two-step admin verification (token + DB role query) |
| Rate Limiting | Multi-tier: IP (30/min), Session (20/min), Global (1000/min); fail-closed on distributed failure |
| CORS | Domain allowlist (no wildcard `*`); separate stricter list for admin endpoints; validated Lovable preview domains |
| Input Sanitization | Null byte, control char, zero-width char removal; prototype pollution prevention; operator spam detection; query length limits (400 chars) |
| Error Sanitization | File paths, connection strings, Bearer tokens, API keys, env values stripped from error messages; messages capped at 500 chars |
| SSRF Prevention | Domain allowlist in `firecrawl-scrape` before any URL fetch |
| XSS | No `dangerouslySetInnerHTML`, no `eval()` — React escapes by default |
| SQL Injection | All DB access via Supabase ORM with parameterized queries; no raw SQL string concat |
| Secret Management | No hardcoded secrets; all values via environment variables validated at startup |

### Findings

**Finding 1 — Inconsistent IP extraction** · `supabase/functions/deck-suggest/index.ts:29`

```typescript
// Current — direct x-forwarded-for without trusted-header validation
const clientIp = req.headers.get('x-forwarded-for')?.split(',')[0]?.trim() || 'unknown';
```

The `rateLimit.ts` shared module already validates trusted header chains (Cloudflare → Fly.io → x-real-ip) and prevents header spoofing. This function extracts IP manually, bypassing that logic.

**Fix:** Use `resolveRateLimitKey()` from `_shared/rateLimit.ts` for consistency.

---

**Finding 2 — Unguarded `JSON.parse` on AI response** · `supabase/functions/deck-suggest/index.ts:228`

```typescript
const parsed = JSON.parse(toolCall.function.arguments); // no try/catch
```

A malformed AI response would cause an unhandled exception, returning a 500 to the client without structured error info.

**Fix:**
```typescript
let parsed;
try {
  parsed = JSON.parse(toolCall.function.arguments);
} catch {
  return errorResponse('Invalid AI response format', 500, jsonHeaders);
}
```

---

**Finding 3 — Audit log is best-effort** · `supabase/functions/_shared/auth.ts`

`logAuthFailure()` is fire-and-forget. Failed auth events may be silently dropped if the insert errors.

**Recommendation:** Persist audit events to a dedicated table with a retry mechanism, or at minimum surface errors to an alerting channel.

---

**Finding 4 — `buildClientFallbackQuery` missing length guard** · `src/lib/search/fallback.ts:207-209`

Empty string is checked but very long inputs (100k+ chars) are not. The mana-pattern regex at line 239 could exhibit ReDoS behavior on adversarially crafted input from the client side.

**Fix:** Add `if (naturalQuery.length > 1000) return '';` at the start of the function.

### Code Duplication

**`src/lib/deckbuilder/decklist-formatters.ts:24-26, 59-61, 99-101`**
Identical card category filtering (mainboard/commanders/companions/sideboard) repeated in three export functions (`buildDecklistText`, `buildArenaText`, `buildMtgoText`). Extract to a shared `groupCardsByCategory(cards)` helper.

**`src/lib/deckbuilder/decklist-formatters.ts:36, 45, 78, 110, 117`**
`.sort((a, b) => a.card_name.localeCompare(b.card_name))` written five times. Extract to a module-level `byCardName` comparator constant.

**`src/lib/search/fallback.ts:220-287`**
Eight near-identical extraction loops for guilds, colors, types, formats, and keywords. Each dynamically creates `new RegExp()`. Consolidate into a single parameterized loop driven by a config object and pre-compile regex at module load.

**`src/hooks/useDeck.ts:131-134, 138-139, 154-155, 159`**
Query key construction and `queryClient.invalidateQueries` pattern repeated across four mutations. Extract to a `invalidateDeckQueries(deckId)` helper.

---

### Overly Complex Functions

**`src/hooks/useSearchHandler.ts:111-310`** — `handleSearch` is 199 lines covering:
- Request deduplication token management
- Timeout orchestration
- Fallback query construction
- Error classification (rate limit vs. network vs. parse)

Split into `executeSearch()`, `classifySearchError()`, `buildFallbackQuery()`.

**`src/lib/search/fallback.ts:207-328`** — `buildClientFallbackQuery` is 121 lines with 10 sequential processing steps. Extract groups of steps into named sub-functions.

**`src/hooks/useSearch.ts:74-413`** — 339 lines managing 15+ state variables, URL sync, IntersectionObserver, and 6 callbacks. Split into:
- `useInfiniteScroll()`
- `useFilterUrlSync()`
- `useSearchState()`

---

### Missing Error Handling (Critical)

**`src/hooks/useSearchQuery.ts:46-62, 138-147`** — `sessionStorage.setItem()` / `getItem()` calls lack try/catch. Storage can throw `QuotaExceededError` (private browsing, full storage).

```typescript
// Fix
try {
  sessionStorage.setItem(key, value);
} catch {
  // Storage unavailable — degrade gracefully
}
```

**`src/hooks/useDeckActions.ts:35-37`** — Supabase query in undo handler has no error handling. If the query fails, the undo operation crashes silently.

**`src/hooks/useRealtimeCache.ts:40-71`** — `payload.new as CachePayload` is an unsafe type assertion on live WebSocket data. Add a runtime type guard:

```typescript
function isCachePayload(v: unknown): v is CachePayload {
  return typeof v === 'object' && v !== null && 'key' in v;
}
```

**`src/hooks/useAuth.ts:76-80`** — `data.user.identities?.length` is accessed without a null check on `data.user`. If signup partially fails, this throws.

---

### Performance Issues

**`src/lib/search/fallback.ts:254, 263, 272, 281, 290, 300`** — `new RegExp()` constructed on every `buildClientFallbackQuery()` call. Pre-compile at module level:

```typescript
const GUILD_RE = /\b(azorius|dimir|...)\b/gi; // module top-level
```

**`src/hooks/useSearch.ts:116-129`** — Six sequential `setState` calls triggered by a URL param change cause six re-renders. Consolidate into a single `useReducer` dispatch.

**`src/hooks/useSearchQuery.ts:18-20`** — Module-level `recentSearches` Map grows indefinitely (only cleaned at 50+ entries) with no max-age on entries. Memory leak on long-lived sessions. Add TTL-based eviction.

---

### TypeScript Type Safety Gaps

**`src/hooks/useAnalytics.ts:266-269`** — Double cast `as unknown as Record<string, unknown>` defeats the type system. Introduce a proper typed overload or union.

**`src/hooks/useRealtimeCache.ts:41-42`** — Unsafe `as CachePayload` cast on WebSocket payload (see error handling section above for fix).

**`src/hooks/useDeck.ts:110, 124-125`** — `Partial<Deck> & { id: string }` allows mutating any column. Define a specific `DeckUpdatePayload` type with only safe-to-update fields.

---

### Race Conditions / Async Bugs (Critical)

**`src/hooks/useSearchHandler.ts:82-86, 170, 207`** — `requestTokenRef` is checked then used in separate operations. A token increment between check and use can apply a stale token to a newer request. Use the token as a captured closure value at the top of the handler, not via ref read.

**`src/hooks/useRealtimeCache.ts:29-84`** — No guard against duplicate channel subscriptions. If the hook mounts twice (StrictMode double-invoke, fast remount), two listeners attach to the same channel. Add a module-level registry or check `supabase.getChannels()`.

**`src/hooks/useAnalytics.ts:140-159`** — Module-level `recentCacheEvents` dedup map is shared across all component instances. Multiple `useAnalytics` hooks interfere with each other's deduplication windows.
