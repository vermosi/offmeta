# Triage Guide

## Label taxonomy
- `bug` — something is broken or incorrect
- `enhancement` — new capability or improvement
- `docs` — documentation-only updates
- `good first issue` — suitable for new contributors
- `help wanted` — maintainer assistance needed
- `security` — security-related items
- `question` — support/usage questions
- `area:ui` — frontend/user interface
- `area:api` — edge functions or external integrations
- `area:infra` — CI/CD, tooling, deployment

## Severity
- **S0**: critical outage or security issue
- **S1**: major functionality broken
- **S2**: degraded experience, partial breakage
- **S3**: minor bug or polish issue

## Triage workflow
1. Confirm reproducibility and collect logs/screenshots.
2. Assign severity and labels.
3. Add "needs-info" if reproduction steps are missing.
4. Link related issues or PRs.
