# Triage Guide

## Labels
- `bug`: Something is broken.
- `enhancement`: Improvement to existing behavior.
- `docs`: Documentation changes.
- `good first issue`: Great for new contributors.
- `help wanted`: Maintainers want assistance.
- `security`: Security-related issues.
- `question`: Support requests.
- `area:ui`: UI/UX surface.
- `area:api`: Edge functions and API requests.
- `area:infra`: CI, tooling, deployment.

## Severity
- **S1 (Critical)**: Outage, data loss, security vulnerability.
- **S2 (High)**: Major feature broken, no workaround.
- **S3 (Medium)**: Partial degradation, workaround exists.
- **S4 (Low)**: Minor bug or cosmetic issue.

## Workflow
1. Apply area + severity labels.
2. Confirm reproduction steps.
3. Assign a maintainer or request help.
4. Add to the next milestone if applicable.
