# Governance

## Roles
- **Maintainers**: Responsible for releases, roadmap, and final review.
- **Contributors**: Submit issues and pull requests.

## Decision making
- Changes are approved by maintainers via PR review.
- For significant changes, consensus is sought in issues or discussions.

## Becoming a maintainer
- Demonstrate sustained, high-quality contributions.
- Show good communication and review practices.
- Maintainers vote to add new maintainers.
